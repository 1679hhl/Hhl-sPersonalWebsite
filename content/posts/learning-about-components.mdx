## Markdown，但更好

你好，当前日期是 <DateTime format="local" />。Markdown 非常适合编写内容，但如果你需要更具交互性的东西怎么办？如果你想提取动态数据、显示格式化的标注或收集用户输入怎么办？这就是组件的用武之地。

---

```graphql
query MyQuery($relativePath: String!) {
  page(relativePath: $relativePath) {
    title
  }
}
```

> 通过组件，你的 Markdown 内容不仅仅是静态文本——它变成了一种动态体验。你可以获取数据、精美地格式化内容，甚至直接在文章中构建交互式元素。

## 为什么在 Markdown 中使用组件？

- **保持内容动态** - 提取实时数据而不是静态文本。
- **提高可读性** - 使用结构化标注、按钮和表单。
- **增强用户交互** - 让用户直接在内容中采取行动。

## 更多使用组件的方式

1. **嵌入 GraphQL 查询** - 动态获取和显示内容。
2. **使用样式化标注** - 使用视觉上独特的元素突出显示关键点。
3. **创建交互式表单** - 让用户注册、提交数据或参与您的内容。
