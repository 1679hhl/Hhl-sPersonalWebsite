      # 这是一个什么项目？


      这是一个启用了 [TinaCMS](https://tina.io) 的 Next.js 应用程序，因此您可以直接在页面上编辑内容。在这个项目中，Tina 基于文件的 CMS 通过 GraphQL 使用：它由 *您* 定义的模式驱动。它不仅从存储库中的 Markdown 文件提供内容，还会自动为您生成 TinaCMS 表单 ✨。


      ## TinaCMS 新手？


      TinaCMS 功能丰富，在配置方式上给用户留出了很大的空间。


      在掌握它是如何协同工作时，可能需要一些学习过程。


      我们文档中的教程将引导您了解基础知识，以及使用该工具的更高级方法。


      查看 [学习指南](https://tina.io/docs/beginner-tutorials/tutorial-overview)，它涵盖了您开始使用 TinaCMS 所需的一切。


      ## 要求


      * Git

      * [Node.js Active LTS](https://nodejs.org/en/about/releases/)

      * NPM, Yarn, pnpm, 或 bun


      ## 安装


      安装项目的依赖项：


      <scriptCopyBlock
        codeLanguage="bash"
        lightTheme="catppuccin-latte"
        darkTheme="catppuccin-mocha"
        commandMap="npm|npm install
        pnpm|pnpm install
        yarn|yarn install
        bun|bun install"
      />


      ## 在本地运行项目


      要运行本地开发服务器：


      <scriptCopyBlock
        codeLanguage="bash"
        lightTheme="catppuccin-latte"
        darkTheme="catppuccin-mocha"
        commandMap="npm|npm run dev
        pnpm|pnpm run dev
        yarn|yarn dev
        bun|bun run dev"
      />


      此命令启动 GraphQL 服务器和 Next.js 应用程序的开发模式。它还会为 TypeScript 和 GraphQL 重新生成您的模式类型，因此对 `/tina/config.ts` 的更改会立即反映出来。


      TinaCloud 内容 API 最有趣的方面之一是，它实际上不需要云端的任何东西即可在本地工作。由于 TinaCMS 默认是基于 Git 的 CMS，因此所有内容都可以通过 CLI 从本地文件系统运行。 😎


      这是开发的理想选择，因为您可以更改内容和代码，并在提交之前查看它们的工作情况。
